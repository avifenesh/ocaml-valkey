Module Valkey.Bloom

Typed helpers for the Valkey Bloom module.

Bloom filters are probabilistic sets: exists returning false means the item is definitely absent, while true means it may be present according to the filter's configured false-positive rate.

Commands route by filter key, so they work correctly in cluster mode as long as applications use the same hash-slot rules as other Valkey key commands.

type scaling =
  1. | Default_scaling
  2. | Expansion of int
  3. | Non_scaling
    (*

    Creation scaling mode. Default_scaling uses the module defaults, Expansion n sends EXPANSION n, and Non_scaling sends NONSCALING.

    *)
type insert_options = {
  1. capacity : int option;
  2. error_rate : float option;
  3. scaling : scaling;
  4. seed : string option;
  5. tightening : float option;
  6. validate_scale_to : int option;
  7. no_create : bool;
}

Options for BF.INSERT. seed is passed through as-is; Valkey validates the module's 32-byte seed requirement. Non_scaling and validate_scale_to are mutually exclusive and raise Invalid_argument before a command is sent.

val add : ?timeout:float -> Client.t -> key:string -> string -> (bool, Connection_error.t) Stdlib.result

BF.ADD. Returns true when the item was newly added and false when it was already present according to the filter.

val madd : ?timeout:float -> Client.t -> key:string -> string list -> (bool list, Connection_error.t) Stdlib.result

BF.MADD. Empty item lists raise Invalid_argument.

val exists : ?timeout:float -> ?read_from:Router.Read_from.t -> Client.t -> key:string -> string -> (bool, Connection_error.t) Stdlib.result

BF.EXISTS. false means definitely absent; true can be a probabilistic positive.

val mexists : ?timeout:float -> ?read_from:Router.Read_from.t -> Client.t -> key:string -> string list -> (bool list, Connection_error.t) Stdlib.result

BF.MEXISTS. Empty item lists raise Invalid_argument.

val insert : ?timeout:float -> ?options:insert_options -> ?items:string list -> Client.t -> key:string -> (bool list, Connection_error.t) Stdlib.result

BF.INSERT. When items is omitted or empty, the command creates or validates the filter and returns an empty list.

val card : ?timeout:float -> ?read_from:Router.Read_from.t -> Client.t -> key:string -> (int, Connection_error.t) Stdlib.result

BF.CARD. Missing filters return 0.

type info = {
  1. capacity : int;
  2. size : int;
  3. filters : int;
  4. items : int;
  5. error_rate : float;
  6. expansion : int option;
  7. tightening : float option;
  8. max_scaled_capacity : int option;
  9. raw : (string * Resp3.t) list;
}

Parsed BF.INFO payload. Fields that do not apply to non-scaling filters are None. raw preserves the full server payload for forward compatibility with newer valkey-bloom versions.
